Work on a different branch to avoid conflicts with other code contributors.

1. Make sure you're in the right folder

If your prototype is called remote-lab for example, use the terminal to navigate to that folder. To do this, run:

cd remote-lab

2. Check what branch you're currently working on

Run this command:

git status

You'll know if you're on the main branch because it will be called master or main. If you're working on either of these you're ok to create a new branch. If you're not, you're already working on another branch.

3. Switch to a new branch

Run this command:

git checkout -b branch-name

Change branch-name to what you want your new branch to be called.

You're now working from a new branch.

4. Open your text editor

Make your code changes.

5. Go back to the terminal once you've made your changes

Run this command:

git status

The terminal can tell that you've made code changes.

6. Stage your changes

Run this command:

git add .

7. Commit your changes

Run this command:

git commit -m "Code changes"

Just change "Code changes" to what you want your commit message to be.

8. Push your changes

Run this command:

git push --set-upstream origin branch-name

Just change branch-name to what you named your branch.

Your changes are now committed and ready to be merged.
